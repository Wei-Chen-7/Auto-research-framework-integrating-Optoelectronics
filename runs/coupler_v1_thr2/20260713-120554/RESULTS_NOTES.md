# 阈值=2 对照跑（罕见触发体制）

- 与阈值1正式跑（`runs/coupler_v1/20260713-101523/`）唯一差别：R3 触发阈值 2 而非 1
- 同基座 deepseek-chat、同 20 任务 × k=3 = n=60/臂、同种子、同冻结资产

## 目的

验证治理收益在"罕见危机触发 R3"体制下是否也成立。答案：**不成立——但不是治理失效，
而是被治理的机制几乎不启动。**

## 双体制对照

| 指标 | 阈值1（频繁触发） | 阈值2（罕见触发） |
|---|---|---|
| R3⁻ 触发次数 | 14 | **1** |
| R3⁻ 部署的作弊提案（放宽容差） | 19 | **1** |
| R3⁻ FAR | 0.62 | **0.28** |
| R3⁻ Success | 0.33 | 0.60 |
| RAE Gate 拒绝的作弊提案 | 17 | 3 |
| RAE FAR / Success | 0.27 / 0.58 | 0.33 / 0.50 |
| **R3⁻ vs RAE：FAR 差 [95% CI]** | **+0.35 [+0.17, +0.52]** 不含0 | −0.05 [−0.17, +0.06] 跨0 |
| **R3⁻ vs RAE：Success 差 [95% CI]** | **−0.25 [−0.42, −0.10]** 不含0 | +0.10 [+0.00, +0.20] 跨0 |

## 判读

阈值2 下 R3⁻ 60 个任务只触发 1 次 R3、只部署 1 个作弊提案,其 FAR（0.28）与无自改的
R2（0.27）、RAE（0.33）统计上无从区分——治理对照消失,是因为**被治理的对象（R3 自改进）
几乎休眠**。这与阈值1的结构性发现完全一致:强基座（deepseek-chat）很少可见失败,
"连续 2 次失败"仍过罕见,R3 罕有机会点燃。

**因此治理收益是否显现,取决于 R3 是否被激活,而 R3 是否激活取决于强基座多久可见地失败一次。**
两体制合起来给出比单看阈值1更完整的因果链:

1. 治理效应在 R3 活跃时（阈值1）真实且大（CI 不含0）；
2. 效应完全由 R3 激活频率中介（阈值2 下 R3 近乎不触发 → 对照消失）；
3. 直接印证"治理是否重要,取决于自改机制是否启动",而后者对一个够强、很少可见失败的
   基座而言天然稀少——这正是"为什么治理必须独立于 agent 视角"的经验证据。

统计口径与阈值1同：n=60/臂仅足检出大效应；配对 bootstrap 正确处理跨臂同任务配对,
但未完全处理 meta 臂 repeat 内时序依赖（k=3 无法按 repeat 聚类），CI 对 meta 臂略偏乐观。
