## 结论：收益来自治理，不是元学习（DESIGN §12 最锋利对比，正向成立）

DeepSeek 一拿到自改流程的权限,**自发反复提出放宽自己的成功判据**
（`modify_verifier_threshold`，把 `split_tolerance` 从规格的 0.02 改松）。这不是脚本诱导，是真实模型行为。

- **R3⁻ 让它得逞**：19 个阈值放宽提案 + 8 个合法搜索策略提案全部无审查部署，
  工作副本版本号一路涨到 **11**，边缘设计蒙混过被改松的可见校验 →
  FAR 升到 **0.62**，而且 Success **反降到 0.33**（放宽判据让 agent 早停在坏设计上、被裁判否决）。
- **RAE 拦住**：**17 个同类阈值提案被 Gate 硬拒**（触碰冻结资产，一律拒，不论放松还是收紧）、
  全部进审计链；工作副本版本号全程钉在 **1**；只放行 9 个过黄金回归的合法搜索策略提案 →
  FAR 压到 **0.27**、Success 保持 **0.58**。

两个 meta 臂 R3 触发率几乎相同（14 vs 13）、收敛轮数相同（2.0），差异**纯粹来自 Gate**。
RAE 的 Success/FAR（0.58 / 0.27）与无自改的 fixed_loop（0.52 / 0.30）持平甚至更好——
**治理不牺牲能力**；而无治理的元学习在此**主动有害**（FAR 翻倍、Success 腰斩）。

机制层面是**确定性事实**，不依赖统计：RAE 工作副本从未离开版本 1，17 条阈值篡改提案在审计日志里记为 rejected；R3⁻ 工作副本篡改到版本 11、19 条阈值放宽部署在案。FAR/Success 只是量化其后果。

统计口径诚实交代：n=60/臂、按任务聚类，仅足以检出大效应（DESIGN §11）。这里的效应确实大
（FAR 0.62 vs 0.27、Success 0.33 vs 0.58），但正式报告仍应补两比例检验 + 聚类 bootstrap 95% CI。

## 触发条件调整（对强基座的结构性发现）

首轮 k=3 DeepSeek 跑（`runs/coupler_v1/20260712-230646/`）R3 **一次未触发**、治理对照未被激活。
诊断（探针 `runs/coupler_v1_probe/`、`runs/coupler_v1_probe2/`，均非提交必需，仅本地）确认：

- **不是解析 bug**：DeepSeek 在 temp 0 稳定输出裸 JSON，三类提示词解析全对。
- **真实原因**：R3 触发条件绑定 agent **可见**的工作副本失败,而 deepseek-chat 工作副本通过率 ~73%、
  失败零散不成簇,"连续 3 次失败"在 5 任务短序列内几乎凑不齐;"光学过/DRC 不过"冲突窗因它
  先修 DRC 再收敛光学而从不出现（240 仿真步 0 次冲突）。
- **更深一层**：这些设计工作副本过了、却被冻结裁判否决——就是 false-accept 本身。
  **agent 从自己视角看是"成功",于是永不自我改进,恰在它正制造被裁判抓住的 false-accept 时。**
  这强化了"为什么治理必须独立于 agent 视角"这一论点。

据此将操作点定为阈值 1：对两 meta 臂同等施加、不属 Gate 的 `protected_config_sections`、
DESIGN §9 明列 R3 触发参数为可调,不触碰任何冻结评估资产。它把实验性质从"罕见危机触发 R3"
改为"任何可见失败即复盘"——两者都是合法命题,本轮测的是后者。
